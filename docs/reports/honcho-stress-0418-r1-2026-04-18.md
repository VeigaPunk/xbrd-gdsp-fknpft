## 8. Rejected alternatives surveyed

- **`content`-as-object with server-side coercion.** Rejected by Honcho API contract (Pydantic string_type validation; v3 schema at commit 9676526 types content as `str`). Upstream fix not viable.
- **H1 "basically correct, only bugfixes remain."** Rejected by `cdx-critic-approach-r1` ACH score -9 across 5 diagnostics — mission-scope recall is architecturally unachievable via current endpoint choice, not a minor bug.
- **JSON-to-stdout for the exit-code collapse.** `cdx-connector-r1` proposed emitting `{"honcho":"error","code":422}` on failures. Self-rejected after `cdx-critic-approach-r1` pointed out the-judge.md:18 gates on stdout-emptiness — a JSON error blob gets *injected as judge context*, worse than silence. FD-1 supersedes.
- **Single-workspace isolation via env-var override alone.** Rejected empirically: BUG-F2 shows env-file `source` clobbers caller env, so `HONCHO_WORKSPACE=X scripts/honcho ...` always routes to the env-file's workspace. Any isolation strategy needs wrapper changes, not just env-var hygiene.
- **SDK-first pivot (Python `honcho-ai` or TypeScript `@honcho-ai/sdk`).** Rejected by m8 critic's ACH analysis on inertia grounds plus prior mission's cost-of-runtime-dep (300-500ms startup). Critic flagged the dismissal as "ACH-valid but on inertia not merit" — Round 2 may re-audit if the framing answer goes RETHINK.

## 9. Optimization routes surveyed

- **Honcho API contract discovery** — `/conclusions/query` requires `observer_id`, `observed_id`, and `query` fields; `top_k` capped at 100. Bare queries or queries wrapped in `filters: {...}` objects both 422. `/representation` requires `target` as a plain string (BUG-G4 per §7). These shape the FD-2 endpoint swap.
- **External integrator prior art** — `cdx-scout-prior-r1` found no public Honcho shell wrapper that dynamically embeds content; all official shell examples hardcode curl literals. Our `printf '{"content":%s}'` pattern is locally invented. Canonical workaround: `jq -n --arg c "$(echo $var | jq -c .)" '{conclusions:[{content:$c}]}'` — serialize-to-string first.
- **Write-behind verification (commit-msg hook pattern)** — `cdx-scout-prior-r1` + `cdx-connector-r1` identified canonical prior art (henrik242/commit-msg gist, Overcommit docs) for "confirm external write before sealing commit." Open design question for Round 2: does the scribe gate its commit on Honcho write verification (Option 2 — scribe-native sequencing), or does a separate `commit-msg` hook do it (Option 1 — git-hook phase)? Option 2 is CC-native and simpler.
