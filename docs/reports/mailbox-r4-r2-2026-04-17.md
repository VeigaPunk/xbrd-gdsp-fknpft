## 4. Per-finding detail

### M4 — COMPACT_PENDING counter-leak (HIGH → SHIPPED)

**Claim.** `compact_events` increments `COMPACT_PENDING` before spawning the worker but previously decremented only inside the worker body. If the worker thread exited abnormally (panic caught by `catch_unwind`) the `fetch_sub` decrement was skipped. `__wait_compact_idle` spins on `COMPACT_PENDING > 0`, so a single counter-leak causes permanent deadlock on the next `compact_idle` call.

**Confidence.** HIGH (mutation kill confirmed; test added; reviewer sign-off).

**Evidence verbatim.**
- `src/mailbox.rs`: unconditional `fetch_sub` now executes in all compact worker exit paths including the `catch_unwind` recovery arm.
- Mutation: counter-leak mutation killed by new test (1/1 for this mutation class).
- `8802cf9` — 188 LOC hardening commit; test suite 21/21 green; clippy clean; fmt clean.
- cdx-reviewer-signoff-r2: PASS on M4 fix; no objections.

**Linchpin.** At bytes-floor cadence (M2), compact triggers 10–100× more frequently. Any panic during compact worker at high cadence makes the leak near-certain, deadlocking all subsequent `compact_idle` callers. M4 must remain shipped before M2 proceeds.

**Verdict.** SHIPPED (commit `8802cf9`; 21/21 green).

### M5 — same-PID compact collision (HIGH → SHIPPED, R3 subprocess test gap)

**Claim.** Worker thread and sync-fallback path both used `std::process::id()` as the sole compact sidecar suffix. Two concurrent compact calls in the same process generated the same filename — second `rename(2)` clobbered the first sidecar silently. Fix: static `AtomicU64` seq suffix incremented per compact call, guaranteeing unique filenames within a process across worker + sync fallback paths.

**Confidence.** HIGH (4-source R1 convergence; seq-suffix fix shipped; R3 subprocess test gap remains).

**Evidence verbatim.**
- `src/mailbox.rs`: `AtomicU64` seq suffix — unique compact sidecar filename per call.
- Mutation-verify-r2: constant-PID mutation (single-process test) not killed — test exercises single-process uniqueness but not cross-process collision (subprocess test deferred R3; see §9 item 1).
- `8802cf9` — 188 LOC; 21/21 green.
- cdx-reviewer-signoff-r2: PASS with note: subprocess test (std::process::Command spawning separate xbreed instances) needed to close Scenario 1 mutation gap.
- R1 severity: downgraded BLOCKER→HIGH after mutation evidence showed Scenario 1 partial rename exclusion. R2: HIGH maintained; seq suffix closes Scenario 2.

**Linchpin.** Scenario 2 (same-process worker + sync fallback racing `events.compact.<pid>`) was the live HIGH path at `sync_channel(1)` cap. Seq suffix closes it. Scenario 1 subprocess gap is a test coverage issue, not a production regression (rename exclusion in `collect_compact_sidecars` still provides partial protection).

**Verdict.** SHIPPED (commit `8802cf9`; subprocess test R3 gap noted; mutation survivor is coverage-only).

### M6 — IO-error silent swallow (HIGH → SHIPPED, io_error_count counter R3)

**Confidence.** HIGH (fix shipped; unified with M4 into single explicit match; R3 observability counter deferred).

**Evidence verbatim.**

- `8802cf9` — 21/21 green.
- cdx-reviewer-signoff-r2: PASS; io_error_count counter + test-observable assertion flagged as R3 hardening.

**Linchpin.** Silent I/O error swallow at bytes-floor cadence (M2) means a recurring disk error would grow the live mailbox unbounded without triggering any alert path. Fix is the load-bearing correctness change; counter is observability enhancement.

### M9 — bimodal rename race (NEW DISCOVERY)

**Source.** ccs-labrat-cadence-redo during M2 bench redo.

**Claim.** Under Option E, the async compact worker and the drain path race to rename `events.compact.<pid+seq>` into the canonical sidecar location. The worker wins approximately 50% of renames (non-deterministic depending on OS scheduler). This creates a bimodal drain-time distribution: "compact-wins path" (fast drain, small live mailbox) and "compact-loses path" (full drain, large live mailbox). A naive drain p50 measurement mixes both populations — the fast half of the distribution masks the slow half, making the p50 appear artificially stable.

**Confidence.** HIGH (bimodal distribution directly observed in cadence bench; methodology implication confirmed by cco-critic-r2 + distiller).

**Evidence verbatim.**
- `b6de41e` — cadence diagnostic harness; baseline.json M2 INCONCLUSIVE data.
- Labrat observation: fast empty-drain (compact-wins) bimodal p50 — p50 masks signal; distribution not unimodal at bytes-floor cadence.
- cco-critic-r2 confirmation: bimodal distribution means p50 is not a reliable aggregate for cadence comparison; path decomposition or single-threaded isolation required.
- M9 methodology rule (distiller synthesis): **decompose drain paths or measure in single-threaded isolation**. Mixed-path p50 is an invalid cadence gate.

**Linchpin.** M9 is the root cause of M2 being UNMEASURABLE-BY-DESIGN. Without path decomposition, any bench comparing baseline vs bytes-floor cadence conflates two populations with different means. The compact-wins path always wins on p50 even without a real cadence improvement — noise masquerades as signal. R3 bench redesign must decompose or isolate.

**Verdict.** NEW DISCOVERY (R3 redesign required; methodology rule: decompose paths or single-threaded isolation). See §9 item 3.

**Proposed.** Remove the sync fallback path from `compact_events` entirely to eliminate the Scenario 2 collision surface. Channel Full → drop instead of sync fallback.

Drop-on-full would require a separate growth bound or counter. Seq suffix (M5) closes the collision with minimal surface change; removal deferred pending M2 cadence redesign.
