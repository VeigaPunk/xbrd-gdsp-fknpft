### Test 1: `compact_keeps_event_at_exact_cutoff_boundary` (mailbox.rs:1260–1290)

**Kills:** mailbox.rs:382 `>=` → `>` mutation survivor from R1-M19.

**Mechanism:** Writes a single event with `timestamp_ms = 0` (the minimum representable timestamp). Calls `compact_events_sync` with `max_age_secs = 2_000_000_000` (2 billion seconds). Since current time in ms (~1.745e12) minus 2_000_000_000_000ms saturates to 0, the cutoff is epoch 0. Under correct `>=` semantics: `0 >= 0` → TRUE → event is KEPT. Under mutated `>` semantics: `0 > 0` → FALSE → event is compacted into digest (lost). Test asserts the event is present in `drain_events` output.

**Red evidence (mutation active):** Test FAIL — `events` did not contain event with `event_type == "boundary"`. Assertion message: _"event at exact cutoff boundary must be kept by >= semantics; got: []"_

**Green evidence (mutation reverted / `>=` correct):** Test PASS.

```rust
// src/mailbox.rs:1259-1290 (verbatim)
#[test]
fn compact_keeps_event_at_exact_cutoff_boundary() {
    let dir = tempdir().unwrap();
    let path = mailbox_path(dir.path());
    std::fs::create_dir_all(path.parent().unwrap()).unwrap();
    let event = Event {
        timestamp_ms: 0,
        from: "boundary-probe".to_string(),
        event_type: "boundary".to_string(),
        payload: "at-exact-cutoff".to_string(),
    };
    // ... writes event, calls compact_events_sync(..., 2_000_000_000),
    //     asserts drain_events returns event
    compact_events_sync(dir.path(), &[], 2_000_000_000).unwrap();
    let events = drain_events(dir.path()).unwrap();
    assert!(
        events.iter().any(|e| e.event_type == "boundary" && e.from == "boundary-probe"),
        "event at exact cutoff boundary must be kept by >= semantics; got: {:?}", events
    );
}
```
